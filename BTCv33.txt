docker build --build-arg PQ=true --build-arg CRYPTO_SUITE=mlkem768 -t btcv33 . 

Phase 2 — SOFT enforcement

+BTCV32_PREFERRED_SUITE_ID=2 +BTCV32_AUTO_HARD_PQ_RATIO=0.95 +BTCV32_AUTO_HARD_WINDOW=1000 

Submit governance proposal:

POST /governance/suite-proposal { "proposal_id": "activate-suite-2-v33", "new_suite_id": 2, "old_suite_id": 0, "soft_activation_height": <current_height + 1000>, "hard_cutoff_height": <current_height + 12000>, "overlap_blocks": 10000, "auto_hard_pq_ratio": 0.95, "auto_hard_window_blocks": 1000, "miner_incentive_bps": 10 } 

Phase 3 — HARD enforcement (auto-triggered or manual)

After pq_migration_ratio >= 0.95 for 1,000 consecutive blocks, HARD enforcement fires automatically. You may also trigger it manually:

+BTCV32_STEALTH_PQ_MANDATORY=true +BTCV32_PREFERRED_SUITE_ID=2 +BTCV32_SUPPORTED_SUITE_IDS=1,2,4,5 

Suite 0 (EC-only) and Suite 3 (ML-KEM-512 for value outputs) are deprecated.

HNDL Key Rotation (S4)

High-value UTXOs (>= 1.0 BTC equivalent) must be rotated within 52,560 blocks (~1 year). Use the wallet rotation helper:

python scripts/wallet_upgrade.py --pq --pq-level 768 --rotate-hndl \ --utxo-tx-id <TXID> --utxo-index <IDX> \ --scan-priv $BTCV32_STEALTH_SCAN_PRIV \ --spend-priv $BTCV32_STEALTH_SPEND_PRIV 

Monitor UTXO ages at GET /wallet/hndl-status.
