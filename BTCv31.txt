# BTCv31 Migration Guide One-page checklist for migrating from BTCv30 to BTCv31 PQ-mandatory mode. ## Step 1 — Enable PQ (EC still accepted) Update `.env`: ```diff -BTCV30_STEALTH_PQ=false +BTCV30_STEALTH_PQ=true BTCV30_STEALTH_PQ_MANDATORY=false 

Rebuild with: docker build --build-arg PQ=true --target full -t btcv30:full .

All nodes and wallets should do this simultaneously (coordinate via governance proposal if on a public network).

EC and PQ outputs are now both accepted. New wallets will generate Kyber768 keys.

Step 2 — Generate and back up Kyber keys

Each wallet operator must run:

btcv30-wallet generate-keys --pq 

This creates independent EC and Kyber768 key material. Back up BTCV30_STEALTH_KYBER_PRIV separately from BTCV30_STEALTH_SCAN_PRIV — they must NOT be the same value.

Allow N epochs (recommended: 2 × snapshot interval) for all wallets to upgrade.

Step 3 — Enforce PQ (EC rejected)

After the migration window, update .env:

BTCV30_STEALTH_PQ=true -BTCV30_STEALTH_PQ_MANDATORY=false +BTCV30_STEALTH_PQ_MANDATORY=true 

From this point, EC stealth outputs are rejected at the validator level. Any wallet that has not migrated will be unable to receive stealth payments.
